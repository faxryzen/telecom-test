## Сборка и запуск
Для сборки требуется:
- Компилятор с поддержкой С++17 (GCC, Clang, MSVC)
- CMake 3.10+
- Boost.Test (для unit-тестирования)
#### Linux
```bash
sudo apt update
sudo apt install cmake g++ libboost-all-dev -y

mkdir build && cd build
cmake ..
make
./output simulator_description.txt
```
Чтобы все описания автоматически скопировались в `build`, положите их в `descriptions/` в корне проекта
#### Windows (MSVC)
```shell
mkdir build
cd build
cmake ..
cmake --build . --config Release
cd ..
.\build\Release\output.exe simulator_description.txt
```
## Ввод и вывод

Запуск программы осуществляется через `./output <simulator_description>`, где параметр - текстовый файл с описанием симуляции. Правила написания этого файла:
* На первой строке число `N` – количество комнат в подземелье, не считая нулевую. `N = [1, 255]`
>Программа корректно обработает эту строку только если на ней просто число без дополнительных символов
* На последующих `N + 1` строках описываются комнаты подземелья в следующем формате: `<номер вершины> <список смежных вершин через запятую> <количество iron> <количество gold> <количество gems> <количество exp>`. Количество ресурсов в диапазоне `[0, 255]`
>Программа корректно обработает эти строки, если их будет не более `N + 1`, если на них будут только числа без дополнительных символов, кроме запятых. Если чисел будет больше/меньше чем необходимо, то программа завершится с ошибкой. Каждая комната должна иметь уникальный номер.
* На последней строке выводится количество еды `M = [2, 255]` и целевой ресурс, ценность которого удвоена для данного подземелья `<количество еды> <iron|gold|gems|exp>`
>Программа корректно обработает эту строку, если на ней будет просто число без дополнительных символов и последнее слово на выбор: `<iron|gold|gems|exp>`

Результат выводится в `result.txt` с ошибочной строкой или алгоритмом работы бота
## Симуляция

Передвижения в другую комнату всегда стоит единицу еды. Сбор ресурса в конкретной комнате стоит единицу еды, если в ней уже собирали ресурсы. Каждый ресурс собирается отдельно, но сразу всей пачкой. Если в комнате лежит `10 gems`, то персонаж сразу собирает `10 gems`. 
## Как добавить свой алгоритм?

Создайте `<your_bot>.hpp/<your_bot>.cpp` и поместите в `src/bots/`. Наследуйтесь от класса `Simulator` из `simulator.hpp` и реализуйте алгоритм в методе `void runBot(std::ostream & output)`. После создайте экземпляр вашего класса и вызывайте `runBot()`:
```cpp
terra::Dungeon dungeon(rooms);
AliceBot sim(dungeon, food, needful_result);
sim.runBot(output);
```
Класс `Simulator` **предоставляет методы для обхода комнат и составления своего алгоритма**:
#### Вывод в `std::ostream`

```cpp
void printState(std::ostream & output) const;
```
Вывод состояния бота: `state <номер комнаты> <кол-во iron> <кол-во gold> <кол-во gems> <кол-во exp>`

```cpp
void printResult(std::ostream & output) const;
```
Вывод результата сборов: `result <кол-во iron> <кол-во gold> <кол-во gems> <кол-во exp> <общая ценность>`

```cpp
void printResource(int res, std::ostream & output) const;
```
Вывод ресурса: `<iron|gold|gems|exp>`
#### Еда

```cpp
int getFood() const noexcept;
```
Возвращает текущее количество еды
#### Комнаты

```cpp
int getPos() const noexcept;
```
Возвращает номер текущей комнаты

```cpp
void go(int next, std::ostream & output);
```
Переход в комнату под номером `next`. Генерирует исключение `std::runtime_error`, если для перехода нет еды, или комната `next` не соединена с текущей позицией

```cpp
int getMaxPossibleRooms() const noexcept
```
Возвращает максимально возможное количество комнат. Поскольку бот не должен знать о настоящем количестве комнат, возвращает 256

```cpp
bool isVisited(int room) const
bool isVisible(int room) const
bool isKnown(int room) const
bool isUnknown(int room) const
```
Проверяют состояние комнаты. Комната может иметь 4 состояния: Посещенная, Видимая, Известен только номер, Неизвестная. Каждый метод проверяет конкретно одно состояние

```cpp
std::vector< int > getNeighbors(int room) const;
```
Возвращает массив смежный комнат. Генерирует исключение `std::runtime_error`, если комната не смежная с посещенной и боту по правилам нельзя узнать соседей этой комнаты
#### Ресурсы

```cpp
terra::res_value getPrices() const noexcept
```
Возвращает массив - стоимость ресурсов на данный момент. При создании `Bot` стоимость *необходимого ресурса* удваивается и применяется на данный массив

```cpp
int getNeedfulResource() const noexcept
```
Возвращает индекс ресурса с удвоенной ценностью

```cpp
  bool hasResources(int room) const;
```
Проверяет есть ли в комнате ресурсы. Генерирует исключение `std::runtime_error`, если комната не является посещенной.

```cpp
  bool hasResources() const
```
Проверяет есть ли в текущей комнате ресурсы.

```cpp
  terra::res_value getResources(int room) const;
```
Возвращает массив ресурсов в комнате. Генерирует исключение `std::runtime_error`, если комната не является посещенной.

```cpp
  terra::res_value getResources() const
```
Возвращает массив ресурсов в текущей комнате.

```cpp
  void collectResource(int index, std::ostream & output);
```
Собирает ресурс по индексу. Генерирует исключение `std::runtime_error`, если для сбора нет еды, или ресурса по индексу нет
## Принцип работы алгоритма AliceBot

1. Фаза исследования - выполняется пока еды не станет на половину меньше.
Бот исследует комнаты, переходя в смежные, у которых меньше номер. Если у комнаты нет не посещенных смежных, бот выполняет поиск в ширину и идет к ближайшей не посещенной комнате с наименьшим номером. При этом бот собирает ресурсы только из новых комнат - в процессе ходьбы к ближайшей не посещенной комнате он не собирает ресурсы. Если бот не находит ни одну не посещенную комнату - возвращается в начальную комнату.
2. Фаза возвращения.
Бот ищет кратчайший путь по посещенным комнатам к начальной. Если таких путей несколько, на развилках выбирает комнату с наименьшим номером. Излишки еды бот тратит в каждой комнате на сбор всех оставшихся ресурсов, начиная с более ценных.
### Пример работы алгоритма:

```input
5
0 1,2 0 0 0 0
1 0,3 5 2 1 15
2 0,4 3 2 1 10
3 1,4 1 0 2 40
4 2,5 2 4 0 15
5 4 0 5 4 10
6 gems
```

```output
go 1
state 1 5 2 1 15
collect gems
state 1 5 2 _ 15
go 3
state 3 1 0 2 40
collect gems
state 3 1 0 _ 40
go 4
state 4 2 4 0 15
collect gold
state 4 2 _ 0 15
go 3
state 3 1 0 _ 40
go 1
state 1 5 2 _ 15
go 0
result 0 4 3 0 182
//             ^^^ 4*11 + 3*23*2 = 182
```
